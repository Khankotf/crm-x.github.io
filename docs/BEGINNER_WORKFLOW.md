# Что делать дальше и в каких программах

Это практичная памятка для первой разработки CRM-X.

## 1. VS Code

Здесь мы пишем код.

Что делать:

- открывать папку `Preject_X`;
- редактировать `app.js`, `styles.css`, `index.html`;
- смотреть созданные документы в папке `docs`;
- запускать команды в терминале VS Code.

Полезные команды:

```powershell
cd C:\Users\khank\VSCODE_Projects\Preject_X
node --check app.js
python -m http.server 8080
```

После запуска открыть в браузере:

```text
http://localhost:8080
```

## 2. Браузер

Здесь проверяется сама CRM.

Что проверять:

- вход под `vip / 123456`;
- вход под `leader / 123456`;
- вход под `alina / 123456`;
- создание лида;
- создание сделки из лида;
- карточку сделки;
- товары и партии;
- выбор ID товара из партии;
- канбан;
- таблицы, фильтры, сортировку и группировку;
- отчеты и проваливание в данные;
- адаптивность на узком экране.

## 3. GitHub

Здесь хранится история проекта.

Что делать:

- после рабочей доработки делать commit;
- писать понятные названия commit, например `Add storage adapter` или `Improve deal product flow`;
- не выкладывать пароли, токены, ключи API;
- держать ветку `main` в рабочем состоянии.

Если Git в терминале ругается на `dubious ownership`, можно один раз выполнить:

```powershell
git config --global --add safe.directory C:/Users/khank/VSCODE_Projects
```

## 4. Supabase

Это кандидат на базу данных и авторизацию.

Для первой продаваемой версии Supabase может закрыть сразу несколько задач:

- PostgreSQL-база;
- регистрация и вход пользователей;
- роли;
- хранение данных разных компаний;
- API для frontend;
- резервные копии на платных тарифах.

Что нужно будет сделать:

- создать проект Supabase;
- создать таблицы через миграцию `supabase/migrations/20260519234500_initial_schema.sql`;
- включить Row Level Security;
- настроить правила доступа по `organization_id`;
- подключить CRM-X к Supabase через API.

Где применять миграцию:

- в Supabase Dashboard можно открыть SQL Editor и выполнить SQL из файла миграции;
- позже можно поставить Supabase CLI и применять миграции командой из терминала.

## 5. Vercel или Netlify

Это сервисы для публикации frontend-приложения.

Что сделать:

- подключить GitHub-репозиторий;
- настроить автоматическую публикацию;
- получить публичную ссылку;
- позже подключить домен.

## 6. Платежи

Для монетизации понадобится платежный провайдер.

Возможные варианты:

- ЮKassa;
- CloudPayments;
- Stripe, если доступен для выбранной юрисдикции;
- ручные счета на первом этапе.

Для первого пилота можно начать без автоматических платежей: выставлять счет вручную, а в системе просто ограничивать доступ по статусу подписки.
