# Архитектура CRM-X

Документ фиксирует, как сейчас устроено приложение и куда его развивать.

## Текущая версия

CRM-X пока работает как статическое frontend-приложение:

- `index.html` содержит разметку экранов;
- `styles.css` отвечает за внешний вид и адаптивность;
- `app.js` содержит состояние, демо-данные, бизнес-логику, таблицы, карточки, отчеты, канбан и авторизацию.
- `src/storage.js` содержит слой хранения.

Данные пока хранятся в браузере пользователя.

## Слой хранения

В `src/storage.js` добавлен единый объект `crmStorage`.

Сейчас он внутри использует `localStorage`, но остальной код больше не должен обращаться к `localStorage` напрямую.

Это нужно, чтобы на следующем этапе заменить хранение в браузере на сервер:

```text
сейчас:
экран CRM -> crmStorage -> localStorage

дальше:
экран CRM -> crmStorage/API client -> backend -> PostgreSQL/Supabase
```

## Почему это важно

Без отдельного слоя хранения любая миграция на базу данных превратится в поиск десятков вызовов по файлу.

С отдельным слоем можно постепенно:

- заменить чтение состояния на запрос к API;
- заменить сохранение состояния на отправку изменений на сервер;
- добавить авторизацию;
- разделить данные разных компаний;
- добавить журнал изменений;
- добавить резервные копии.

## Ближайшая техническая цель

Следующий аккуратный шаг: разделить `app.js` на модули.

Рекомендуемые модули:

- `src/storage.js` — слой хранения;
- `src/state.js` — нормализация и демо-данные;
- `src/reports.js` — расчеты отчетов;
- `src/tables.js` — таблицы, фильтры, группировки;
- `src/dialogs.js` — карточки объектов;
- `src/kanban.js` — канбан;
- `src/products.js` — продукты, партии, серийные номера;
- `src/auth.js` — пользователи, роли, вход.

После этого будет проще переносить проект на React/Vite или подключать backend.

## База данных

Первая миграция для Supabase лежит в:

```text
supabase/migrations/20260519234500_initial_schema.sql
```

Она создает каркас многопользовательской CRM:

- организации;
- профили пользователей;
- роли пользователей в организации;
- компании;
- контакты;
- лиды;
- сделки;
- товары и серийные единицы;
- позиции сделок;
- задачи;
- планы продаж;
- рекламные кампании;
- обращения;
- журнал аудита.

В миграции включены базовые политики Row Level Security. Их нужно будет дополнительно проверить на реальном Supabase-проекте перед первым пилотом.

## Supabase Adapter

Файлы подключения:

- `src/supabase-config.js` — URL проекта и публичный `publishable key`;
- `src/supabase-client.js` — создание клиента Supabase;
- `src/data-adapter.js` — общий адаптер данных.

Пока CRM-X работает в гибридной логике:

```text
основное состояние CRM -> local adapter -> localStorage
табличные операции будущего backend -> supabase adapter -> Supabase
```

Такой режим нужен, чтобы переносить сущности постепенно. Сначала можно перенести лиды, потом сделки, контакты, компании, задачи и продукты.
