# Чеклист перед первой продажей CRM-X

Этот список нужен, чтобы не продавать сырой прототип как готовый продукт.

## Продукт

- [ ] Выбрана первая ниша.
- [ ] Понятно, кому продаем CRM-X.
- [ ] Понятно, какую боль решает CRM-X.
- [ ] Демо-данные соответствуют выбранной нише.
- [ ] Есть описание тарифов.
- [ ] Есть короткий сценарий демонстрации.

## Техническая база

- [ ] Код сохранен в GitHub.
- [ ] Есть стабильная версия прототипа.
- [ ] Данные хранятся в базе, а не только в `localStorage`.
- [ ] У всех бизнес-записей есть `organization_id`.
- [ ] Данные разных компаний изолированы.
- [ ] Авторизация настоящая, а не демо-логины.
- [ ] Восстановление пароля работает.
- [ ] Роли работают: менеджер, руководитель, VIP/admin.
- [ ] Приложение открывается по публичной ссылке.
- [ ] Есть резервные копии.

## Основные сценарии CRM

- [ ] Создать лид.
- [ ] Открыть карточку лида.
- [ ] Квалифицировать лид.
- [ ] Дисквалифицировать лид с причиной.
- [ ] Создать сделку из лида.
- [ ] Открыть карточку сделки.
- [ ] Добавить несколько товаров в одну сделку.
- [ ] Выбрать ID товара из партии.
- [ ] Запретить сохранение сделки, если ID товара из партии не выбран.
- [ ] Переместить сделку по канбану.
- [ ] Закрыть сделку как успешную.
- [ ] Проданная единица товара больше не доступна для новой сделки.
- [ ] Создать задачу по сделке.
- [ ] Увидеть задачу на дашборде.

## Отчеты

- [ ] Воронка продаж работает.
- [ ] Выручка план-факт работает.
- [ ] Отчет по продуктам учитывает несколько товаров в одной сделке.
- [ ] ROMI работает.
- [ ] UTM-отчет работает.
- [ ] Отчет по промокодам работает.
- [ ] Отчет по менеджерам работает.
- [ ] Отчеты кликабельны и открывают исходные данные.

## Безопасность данных

- [ ] Данные клиентов не смешиваются между организациями.
- [ ] Важные действия логируются.
- [ ] Есть экспорт данных.
- [ ] Есть импорт контактов, лидов и товаров.
- [ ] Удаление данных контролируется.

## Интерфейс

- [ ] Работает на ноутбуке.
- [ ] Работает на планшете.
- [ ] Работает на смартфоне.
- [ ] Таблицы usable на мобильном.
- [ ] Карточки не вылезают по ширине.
- [ ] Ошибки понятны пользователю.
- [ ] Пустые состояния объясняют, что делать дальше.

## Бизнес и юридическая часть

- [ ] Выбран домен.
- [ ] Есть политика конфиденциальности.
- [ ] Есть оферта или договор.
- [ ] Выбран платежный провайдер.
- [ ] Понятно, кто и как оказывает поддержку.
- [ ] Понятно, что делать при сбое.

## Пилотный запуск

- [ ] Выбран пилотный клиент.
- [ ] Подготовлен импорт данных.
- [ ] Созданы пользователи.
- [ ] Проведено обучение.
- [ ] Создан список обратной связи.
- [ ] Ошибки из пилота приоритизированы.

## Готовность к первой оплате

CRM-X можно продавать первому клиенту, когда:

- хотя бы один пилот прошел полный рабочий сценарий;
- нет известных критичных ошибок с потерей данных;
- логин и разделение данных настоящие;
- ключевые отчеты проверены;
- есть канал поддержки;
- есть понятное коммерческое предложение.
